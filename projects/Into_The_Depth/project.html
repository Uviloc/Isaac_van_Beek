Process

An interesting aspect is the fact that each attack you use on the enemies will use up attack points. These points can only be gotten by picking up items that are located around the maze.
The textures are also my own, they were originally made for a different project, but they fitted perfectly into the game as it was.

Challenges

Some difficulties I came across were the damage taken from the sliding blocks. This damage should only be applied when the player was between a box and a wall. However, many of the solutions I first thought of did not work. Such as having the player check if it's colliding a box and a wall. Then checking on which sides these collisions were on.
This had the issue that I could not then check what force the box was applying to the player. So even when the speed of the box was minimal the player would still receive max damage.
I also at one point tried raytracing, but in the end, I came up with the solution of adding a trigger collider to each side of the box.


Then this single collider can check if both a player and a wall is inside of it.
The collision box was also scaled (by code so it's consistent for any sized box) to only allow a player to be in between it and a wall, so it would only trigger when it was actually fully colliding.

Mobile Development

I came back to this project about a year later.
The idea was always to create a mobile version for this game. The tilting field was directly inspired by the unique controls that phones had.

The process of getting this to work was both easier and more difficult then expected.
I was under the understanding that gyroscopes and accelerometers were difficult to implement as not a lot of games have it as a feature. However, Unity had built-in features to make the use of these mobile components possible.

There were however parts that gave me more trouble then expected.
One was the problems that came with downgrading the graphics and materials, as I was using Substance Designer materials.
The other was how the limits of the rotation needed to be tweaked much more; therefore, the game balancing was also thrown out of place.

Plans for the future

Halfway through this project I decided I wanted to release this project for people to play. This did influence my work, since I needed to have some future proofing and have all assets and audio be correctly licensed.

Then after I finished this project and got my grade, I made a future plan to update the game with more content and improve the core concept. However, with me being busy with my study I knew that this would take a while to come to fruition.
